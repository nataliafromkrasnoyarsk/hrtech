# Create Landing Content

Интерактивный wizard для создания контента посадочных страниц.

## Usage

```
/create-landing [бриф или путь к файлу]
```

## What This Command Does

Создаёт полный контент для лендинга на основе информации о продукте или сервисе:

1. **Собирает информацию** — из брифа, файла или через интерактивные вопросы
2. **Определяет структуру** — тип лендинга и набор блоков
3. **Генерирует контент** — тексты для всех секций
4. **Оптимизирует для SEO** — ключевые слова, meta-теги
5. **Экспортирует результат** — в Markdown, HTML или JSON

## When to Use This Command

- Нужно создать новую посадочную страницу с нуля
- Есть бриф о продукте и нужен готовый контент
- Запускаете новый продукт/сервис и нужен лендинг
- Хотите быстро получить структурированный контент для вёрстки

## Input Options

### 1. Текстовый бриф

```
/create-landing Cloud Storage — S3-совместимое хранилище VK Cloud. Неограниченный объём, 99.99% доступность, от 0.8 руб/ГБ. Для разработчиков и DevOps.
```

### 2. Путь к файлу

```
/create-landing briefs/new-product.md
```

### 3. Интерактивный режим

```
/create-landing
```

Без аргументов команда запускает wizard и задаёт вопросы пошагово.

## Brief Format

Бриф может содержать:

```yaml
# Обязательно
product_name: "Название продукта"
description: "Краткое описание (1-2 предложения)"
key_features:
  - "Преимущество 1"
  - "Преимущество 2"
  - "Преимущество 3"

# Рекомендуется
target_audience: "Целевая аудитория"
competitive_advantages:
  - "Уникальное преимущество 1"
  - "Уникальное преимущество 2"
cta_goals:
  - "Регистрация"
  - "Демо"

# Опционально
pricing_info: "Информация о ценах"
brand_tone: "professional" | "friendly" | "technical"
```

Или в свободной форме:

```
Продукт: Cloud Containers
Описание: Kubernetes как сервис для запуска микросервисов
Преимущества:
- Создание кластера за 10 минут
- Сертификация CNCF
- SLA 99.95%
- Автомасштабирование
Аудитория: DevOps-инженеры и разработчики
```

## Output Formats

### Markdown
Структурированный текст с заголовками и секциями — для копирования и редактирования.

### HTML
Готовая разметка с CSS-классами — для передачи верстальщику.

### JSON
Структурированные данные — для CMS или фронтенд-приложений.

## Quality Standards

Сгенерированный контент соответствует:

- **SEO**: ключевые слова в H1, первом абзаце, meta-тегах
- **Стиль VK Cloud**: активные глаголы, конкретные цифры, без канцеляризмов
- **Структура**: логичная последовательность блоков, иерархия заголовков
- **UX**: понятные CTA, сканируемый текст
