# VK Cloud Landing Page Knowledge

Структура, стиль и правила лендингов VK Cloud на основе анализа реальных страниц.

---

## ТИПОВАЯ СТРУКТУРА ЛЕНДИНГА

### Общая схема

```
+-----------------------------------------------------------+
|  HEADER (навигация)                                       |
+-----------------------------------------------------------+
|  HERO                                                     |
|  H1 + подзаголовок + CTA                                  |
+-----------------------------------------------------------+
|  PROMO BANNER (опционально)                               |
|  Акция или спецпредложение                                |
+-----------------------------------------------------------+
|  ABOUT / VALUE PROPOSITION                                |
|  Краткое описание платформы/сервиса                       |
+-----------------------------------------------------------+
|  FEATURES / BENEFITS                                      |
|  Преимущества (иконки + текст)                            |
+-----------------------------------------------------------+
|  SOLUTIONS / PRODUCTS                                     |
|  Карточки продуктов/решений                               |
+-----------------------------------------------------------+
|  HOW IT WORKS / RELIABILITY                               |
|  Как работает / Надёжность                                |
+-----------------------------------------------------------+
|  INTERFACE PREVIEW (опционально)                          |
|  Скриншоты интерфейса                                     |
+-----------------------------------------------------------+
|  TRUST / CLIENTS                                          |
|  Логотипы клиентов                                        |
+-----------------------------------------------------------+
|  CTA FINAL                                                |
|  Финальный призыв к действию                              |
+-----------------------------------------------------------+
|  FOOTER                                                   |
|  Ссылки, контакты, юридическая информация                 |
+-----------------------------------------------------------+
```

## БЛОКИ ЛЕНДИНГА: ДЕТАЛЬНЫЙ РАЗБОР

### 1. Hero-секция

```yaml
type: hero
elements:
  - H1: Название продукта/платформы
  - Подзаголовок: Ценностное предложение (1-2 предложения)
  - CTA: Кнопка(и) действия
  - Визуал: Иллюстрация или скриншот

examples:
  main_page:
    h1: "VK Cloud"
    subtitle: "Облачная платформа для цифровизации бизнеса"
    cta: ["Попробовать", "Получить консультацию"]

  product_page:
    h1: "Cloud Containers"
    subtitle: "Сервис для работы с кластерами Kubernetes в облаке"
    cta: ["Создать кластер"]

rules:
  - H1 содержит название продукта
  - Подзаголовок объясняет ЧТО ЭТО за 1 предложение
  - Минимум 1 CTA, максимум 2
  - Не более 20 слов в подзаголовке
```

### 2. Промо-баннер

```yaml
type: promo
elements:
  - H2: Название акции
  - Описание: 2-3 предложения
  - CTA: Ссылка на подробности

examples:
  - title: "Облачный апгрейд на 5 млн тенге"
    description: |
      Постройте второе плечо в облаке или перенесите инфраструктуру в VK Cloud.
      Получите до 5 млн тенге бонусами на облачные сервисы.
    cta: "Подробнее"

  - title: "Бесплатный тест"
    description: |
      Легкий старт в облаке с бонусами на 24 000 тенге. Зарегистрируйтесь
      на платформе и получите приветственные бонусы.
    cta: "Узнать подробности"

rules:
  - Конкретная сумма/выгода в заголовке
  - Описание объясняет условия
  - CTA ведёт на детальную страницу
```

### 3. About / Value Proposition

```yaml
type: about
elements:
  - H2: Заголовок секции
  - Описание: 2-4 предложения
  - Feature-список: 3-6 кратких пунктов с иконками

examples:
  title: "Облачная платформа VK Cloud"
  description: |
    Платформа для разработки инновационных продуктов, хранения и анализа данных,
    защиты от атак.
  features:
    - "Решения разработаны экспертами VK (ex-Mail.ru Group)"
    - "Арендуйте виртуальные машины, собирайте и анализируйте Big Data"
    - "Быстро выводите продукты на рынок с Kubernetes"
    - "Используйте Cloud-технологии для современных приложений"

rules:
  - Описание отвечает на "Что это?"
  - Features — это преимущества, не функции
  - Каждый пункт — одно предложение
```

### 4. Features / Benefits

```yaml
type: features
elements:
  - H2: "Почему стоит выбрать..." / "Преимущества"
  - Список: 4-6 пунктов

structure:
  - Заголовок пункта (опционально)
  - Описание (1-2 предложения)

examples:
  title: "Почему стоит выбрать облако"
  items:
    - "Аутсорс ИТ-инфраструктуры: вам не нужно покупать и обслуживать серверы."
    - "Вы делегируете администрирование технологий провайдеру."
    - "PaaS для быстрой разработки и сокращения Time-to-Market."
    - "Запуск виртуальной машины за 5 минут."
    - "Pay-as-you-go: платите только за потребленные ресурсы."

rules:
  - Начинать с выгоды, не с функции
  - Добавлять цифры где возможно
  - Каждый пункт — самодостаточный
```

### 5. Solutions / Products

```yaml
type: solutions
elements:
  - H2: "Решения" / "Сервисы"
  - Карточки продуктов

card_structure:
  - Иконка
  - Название (H3)
  - Краткое описание (1-2 предложения)
  - Ссылка (опционально)

examples:
  - name: "Виртуальные серверы"
    description: "Вычислительные ресурсы преднастроены, можно быстро запустить сайт или приложение."

  - name: "Объектное хранилище"
    description: "Неограниченный объем данных, быстрое масштабирование под потребности бизнеса."

  - name: "Кластеры Kubernetes"
    description: "Развертывание, масштабирование и мониторинг микросервисных приложений."

  - name: "Базы данных в облаке"
    description: "Большой выбор СУБД: MySQL, MongoDB, Redis, ClickHouse, PostgreSQL."

rules:
  - 4-8 карточек на секцию
  - Название — существительное или словосочетание
  - Описание начинается с глагола или существительного
  - Не использовать "позволяет", "даёт возможность"
```

### Правила размещения ключей

```yaml
h1:
  - Основной ключ обязательно
  - Название продукта
  - До 60 символов

h2:
  - Вспомогательные ключи
  - Естественные формулировки
  - Не повторять H1

first_paragraph:
  - Основной ключ в первых 100 символах
  - Естественное вхождение

body:
  - Плотность ключей: 1-3%
  - Равномерное распределение
  - LSI-ключи для разнообразия

meta:
  title:
    - Основной ключ в начале
    - До 60 символов
    - Бренд в конце
  description:
    - Основной + 1-2 дополнительных ключа
    - 120-160 символов
    - CTA или выгода
```

## СТИЛЬ И ТОНАЛЬНОСТЬ

### DO — Используем

```yaml
tone:
  - Профессиональный, но доступный
  - Уверенный без агрессии
  - Конкретный и фактологический

verbs:
  - "Запустите" (не "Вы можете запустить")
  - "Сокращает" (не "Позволяет сократить")
  - "Обеспечивает" (не "Даёт возможность обеспечить")

numbers:
  - "за 5 минут"
  - "SLA 99.95%"
  - "до 60% экономии"
  - "8000+ клиентов"
  - "50+ сервисов"

specifics:
  - Названия технологий: Kubernetes, PostgreSQL, S3
  - Стандарты: CNCF, 152-ФЗ, ФСТЭК
  - Метрики: время, проценты, количество
```

### DON'T — Избегаем

```yaml
vague:
  - "инновационный"
  - "передовой"
  - "уникальный"
  - "лучший"
  - "значительно"

bureaucratic:
  - "осуществляет"
  - "производит"
  - "реализует"
  - "в настоящее время"

passive:
  - "сервис позволяет сделать" -> "сервис делает"
  - "можно использовать" -> "используйте"
  - "появляется возможность" -> "теперь можно"

complex:
  - Предложения >25 слов
  - Причастные обороты >5 слов
  - Двойные отрицания
```

### Примеры трансформации

| Было | Стало |
|------|-------|
| "Позволяет осуществлять развёртывание" | "Развёртывайте" |
| "Обеспечивает возможность масштабирования" | "Масштабируется автоматически" |
| "Значительно сокращает время" | "Сокращает время на 60%" |
| "Инновационная платформа" | "Платформа с 50+ сервисами" |
| "Уникальные возможности" | "Kubernetes с сертификацией CNCF" |

## ШАБЛОНЫ ТЕКСТОВ

### Hero-подзаголовок

```
Формула: [Что это] для [кого] / [главная выгода]

Примеры:
* "Облачная платформа для разработки и масштабирования цифровых продуктов"
* "Kubernetes как сервис для быстрого запуска микросервисов"
* "S3-хранилище для бэкапов, статики и Big Data"
```

### Пункт преимущества

```
Формула: [Выгода]: [как достигается] / [результат с цифрой]

Примеры:
* "Экономия до 40%: платите только за использованные ресурсы"
* "Запуск за 5 минут: выберите конфигурацию и нажмите одну кнопку"
* "Доступность 99.95%: отказоустойчивая архитектура с резервированием"
```

### Описание продукта

```
Формула: [Название] — [что это] для [задача]. [Ключевая характеристика].

Примеры:
* "Cloud Containers — Kubernetes как сервис для запуска микросервисов. Сертификация CNCF, SLA 99.95%."
* "Cloud Storage — S3-совместимое хранилище для любых объёмов данных. Доступность 99.99%, от 0.8 руб/ГБ."
```

## ЧЕКЛИСТ ОБНОВЛЕНИЯ ЛЕНДИНГА

### Перед обновлением

```yaml
before_update:
  - [ ] Сохранить исходную структуру
  - [ ] Определить количество блоков
  - [ ] Подсчитать предложения в каждом блоке
  - [ ] Выделить ключевые элементы (H1, H2, CTA)
```

### SEO-проверка

```yaml
seo_check:
  - [ ] Основной ключ в H1
  - [ ] Ключи в H2 (где уместно)
  - [ ] Ключ в первом абзаце
  - [ ] Плотность ключей 1-3%
  - [ ] Title до 60 символов
  - [ ] Description до 160 символов
```

### Проверка контента

```yaml
content_check:
  - [ ] Нет устаревших данных
  - [ ] Минимум 3 конкретные цифры
  - [ ] Нет канцеляризмов
  - [ ] Нет превосходных степеней без доказательств
  - [ ] CTA понятны и привлекательны
```

### Проверка структуры

```yaml
structure_check:
  - [ ] Количество блоков = исходному
  - [ ] Количество предложений +-1
  - [ ] Иерархия H1->H2->H3 сохранена
  - [ ] Логика повествования не нарушена
```

## ТИПИЧНЫЕ ОШИБКИ

### Ошибки SEO

| Ошибка | Решение |
|--------|---------|
| Ключ только в конце текста | Перенести в H1 и первый абзац |
| Переспам ключей (>3%) | Заменить часть на LSI-синонимы |
| Title >60 символов | Сократить, оставив главное |
| Description без CTA | Добавить призыв или выгоду |

### Ошибки контента

| Ошибка | Решение |
|--------|---------|
| "Позволяет осуществлять" | Заменить на глагол действия |
| Нет цифр | Добавить: время, %, количество |
| Длинные предложения | Разбить на 2-3 коротких |
| Общие фразы без конкретики | Добавить технологии, стандарты |

### Ошибки структуры

| Ошибка | Решение |
|--------|---------|
| Добавлены новые блоки | Удалить, сохранить исходное количество |
| Изменена иерархия H | Восстановить исходную |
| Удалены CTA | Вернуть призывы к действию |

## УСПЕШНЫЕ КЕЙСЫ

Этот раздел содержит примеры удачных генераций, одобренных пользователем.
Используй их как референс для стиля и качества.

### Как добавить новый пример

После успешной генерации агент предлагает сохранить результат как пример.
Формат записи:

```yaml
case:
  id: "{номер}"
  date: "{дата}"
  page_type: "{тип страницы: main/product/promo}"
  product: "{название продукта}"

  context:
    task: "{краткое описание задачи}"
    keywords: ["{ключ1}", "{ключ2}"]
    instructions: "{указания пользователя}"

  before:
    block_type: "{тип блока}"
    text: |
      {исходный текст}

  after:
    text: |
      {обновлённый текст}

  changes:
    - "{что изменено 1}"
    - "{что изменено 2}"

  metrics:
    seo_score: "{было} -> {стало}"
    keywords_added: {число}
    numbers_added: {число}
```

## РЕФЕРЕНС: ОТРАСЛЕВОЙ ЛЕНДИНГ (МЕДИА)

Разбор структуры лендинга отраслевого решения на примере "Облачные решения для медиаконтента".

### Тип лендинга: Industry Solution Landing

Особенности:
- **Фокус на отрасли**, а не на продукте
- **Вызовы отрасли** + ответы облака
- **Детальные сценарии** с Задача/Решение/Сервисы
- **Партнёрские сервисы** в дополнение к собственным
- **Кейсы клиентов отрасли**

### Общая структура (6 секций)

```
┌─────────────────────────────────────────────────────────────┐
│  1. HERO (отраслевой)                                       │
│     Решения для отрасли + подзаголовок                      │
├─────────────────────────────────────────────────────────────┤
│  2. CHALLENGES & SOLUTIONS                                  │
│     6 вызовов отрасли + ответы облака + CTA                 │
├─────────────────────────────────────────────────────────────┤
│  3. SERVICES GRID (отраслевой)                              │
│     10 сервисов (8 VK Cloud + 2 партнёрских) + CTA          │
├─────────────────────────────────────────────────────────────┤
│  4. USE CASES DETAILED                                      │
│     9 сценариев: категории + Задача/Решение/Сервисы + CTA   │
├─────────────────────────────────────────────────────────────┤
│  5. PROMO / TRIAL                                           │
│     Бонус на тестирование + 2 CTA                           │
├─────────────────────────────────────────────────────────────┤
│  6. CLIENTS CASES                                           │
│     7 кейсов клиентов отрасли                               │
└─────────────────────────────────────────────────────────────┘
```

