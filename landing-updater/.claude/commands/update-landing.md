# /update-landing

Запусти агента `landing-updater-wizard` для обновления текста лендинга.

## Контекст задачи

Пользователь хочет обновить текст лендинга VK Cloud. Твоя задача:

1. Получить контент (URL, текст или файл)
2. Разобрать на блоки с сохранением структуры
3. Собрать SEO-параметры
4. Учесть референсы и указания
5. Проанализировать и предложить улучшения
6. Сгенерировать обновлённый текст

## Аргументы

Если пользователь передал URL:
```
$ARGUMENTS содержит URL страницы для обновления
```

## Инструкции

### Если передан URL

1. Используй WebFetch для загрузки страницы:
   ```
   WebFetch url="$ARGUMENTS" prompt="Извлеки весь текстовый контент: заголовки H1/H2/H3, основной текст, CTA-кнопки, списки. Верни структурированный результат."
   ```

2. После загрузки покажи структуру и продолжи workflow агента

### Если URL не передан

Начни диалог с приветствия и запроси способ ввода контента.

## Workflow

```
1. ТЕКСТ         URL, текст или файл
       |
       v
2. БЛОКИ         Автоматическая сегментация
       |
       v
3. SEO           Ключевые слова, title, description
       |
       v
4. РЕФЕРЕНСЫ     Страница-образец или указания
       |
       v
5. АНАЛИЗ        Проблемы и предложения по блокам
       |
       v
6. ГЕНЕРАЦИЯ     Обновлённый текст
       |
       v
7. СОХРАНЕНИЕ    Оценка и сохранение удачного примера
```

## Примеры использования

### По ссылке
```
/update-landing https://vkcloud.kz/containers/
```

### Интерактивно
```
/update-landing
```

### С текстом в сообщении
```
Обнови этот текст лендинга:

# VK Cloud
Облачная платформа для цифровизации бизнеса...
```

## Связанные ресурсы

- Агент: `landing-updater-wizard`
- База знаний: `landing-knowledge` (skill)

## Ключевые принципы

```
СОХРАНЯЕМ:                 ОБНОВЛЯЕМ:
- Количество блоков        - Формулировки
- Количество предложений   - Актуальность данных
- Иерархию H1->H2->текст   - SEO-оптимизацию
- Общий смысл              - Устаревшие термины
```

## Новые возможности

### Сохранение удачных примеров

После генерации агент предложит оценить результат:
- **Отлично** — сохранить как пример для будущих генераций
- **Хорошо** — использовать, но не сохранять
- **Нужны правки** — указать что исправить

Удачные примеры сохраняются в skill `landing-knowledge` и используются как референс.

### Команды сохранения

| Команда | Действие |
|---------|----------|
| `сохранить пример` | Сохранить результат как успешный кейс |
| `сохранить блок N` | Сохранить конкретный блок как пример |
