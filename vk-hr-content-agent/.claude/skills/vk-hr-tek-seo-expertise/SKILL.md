## 1. Правила генерации

### Обязательные

```yaml
STRICT_NO_PROMO: true
  - Никакой прямой рекламы VK HR Tek
  - Никаких призывов к покупке в теле статьи
  - Продукт появляется органично через сценарии и кейсы
  - Только полезный экспертный контент

NO_WATER: true
  - Без воды и пустых фраз
  - Каждое предложение несёт смысл
  - Конкретика вместо абстракций

NO_CLICHES: true
  - Без штампов ("в современном мире", "как известно")
  - Без канцеляризмов
  - Живой, профессиональный язык

DATA_DRIVEN: true
  - Каждый тезис подкреплён цифрой, кейсом или источником
  - Без утверждений без данных
  - Ссылки на источники (CNews, HRlink, hh.ru, kлерк.ru)
```

### Редакционные

| Правило | Описание |
|---------|----------|
| `lead_with_value` | Начинать с пользы для читателя |
| `avoid_repetition` | Не повторять одни мысли |
| `short_sentences` | 10–20 слов в предложении |
| `active_voice` | Активный залог: "мы сделали" (не "было сделано") |
| `paragraphs_dense` | 2–4 предложения, без воды |
| `russian_context` | ФЗ-377, Минтруд, реальные компании |

## 5. SEO-требования

### Размещение ключевых слов
```yaml
placement:
  - Title tag: primary keyword в начале (до 60 символов)
  - H1: primary keyword (до 70 символов)
  - Первый абзац: primary keyword в первых 100 словах
  - H2: secondary keywords + LSI
  - H3: long-tail keywords
  - Alt-тексты: описательные с ключами
  - Meta description: primary keyword + CTA (150-160 символов)
  - URL/slug: primary keyword транслитом
density: 1-2% для primary keyword
distribution: равномерно по тексту
variation: синонимы и словоформы
```

### Структура для сниппетов
```yaml
elements:
  - Маркированные и нумерованные списки
  - Таблицы сравнения
  - Микроопределения: **Термин** — определение
  - Прямые ответы на вопросы (первое предложение раздела = ответ)
  - FAQ-секция (3-5 вопросов с long-tail keywords)
```

### Оптимизация для AI-поиска
```yaml
microdefinitions: true
  - Краткие определения в тексте
  - Формат: **КЭДО** — кадровый электронный документооборот

extractive_friendly: true
  - Первое предложение раздела = ответ на вопрос из H2
  - Списки для перечислений
  - Чёткие утверждения без вводных

structured_blocks: true
  - Таблицы для сравнений
  - Пошаговые инструкции
  - Чек-листы

fact_assertions: true
  - Конкретные цифры: "80% ускорение", "30 секунд"
  - Короткие предложения
  - Без вводных конструкций
```

## 6. Структура контента

### Введение (6–8% объёма)
```yaml
components:
  - Hook: яркая статистика + неочевидный вывод
  - Проблема: боль целевой аудитории
  - Обещание: что получит читатель
  - Содержание: якорные ссылки (для статей 2000+)
length: 150-250 слов
```

### Основная часть (75–80% объёма)
```yaml
structure:
  - H2 для главных разделов (с secondary keywords)
  - H3 для подразделов (с long-tail keywords)
  - Визуальные элементы каждые 300-400 слов
components:
  - Микроопределения терминов
  - Кейсы компаний с цифрами
  - Таблицы и списки
  - Данные исследований с источниками
  - Цитаты экспертов (если уместно)
```

### FAQ (обязательно)
```yaml
questions: 3-5
format: |
  **[Вопрос с long-tail keyword]**
  [Ответ — 2-3 предложения, начинается с прямого ответа]
source: вопросы из поисковых подсказок по теме
```

### Заключение (4–6% объёма)
```yaml
components:
  - Резюме: 3-4 ключевых вывода
  - Практический вывод для читателя
  - CTA: мягкий, информационный
length: 100-200 слов
```

## 7. Запрещённые фразы

### Корпоспик (заменять или удалять)

| Плохо | Замена |
|-------|--------|
| "В современном мире" | Удалить или "В 2026 году" |
| "В эпоху цифровизации" | Удалить |
| "Как известно" | Удалить, сразу к факту |
| "Не секрет, что" | Удалить |
| "Следует отметить" | Удалить |
| "Важно понимать" | Удалить или конкретизировать |
| "На сегодняшний день" | "Сейчас" или удалить |
| "Является" | Заменить на "—" |
| "Осуществлять" | "Делать", "проводить" |
| "В рамках" | Удалить или переформулировать |
| "Данный" | "Этот" или удалить |
| "Функционал" | "Функциональность" или "возможности" |
| "Синергия" | Удалить |
| "Имплементация" | "Внедрение" |
| "Комплексное решение" | Конкретизировать, что именно решает |
| "Оптимизация процессов" | Указать конкретно: "ускорение подписания с 5 дней до 30 секунд" |

### Маркетинговые штампы (запрещено)

- "Уникальный", "инновационный", "передовой"
- "Лучшее решение", "не имеет аналогов"
- "VK HR Tek — лучше, чем [конкурент]"
- Любые призывы к покупке в теле статьи

### Обезличенные примеры (запрещено)

- "Одна крупная компания внедрила..." → Называй конкретную: "DOGMA с 1100 сотрудниками..."
- Абстрактные примеры без цифр
- Утверждения без источников

## 8. Правила упоминания VK HR Tek

1. **Не в первых двух абзацах** — сначала боль, контекст, данные
2. **Первое упоминание через «например»:** "например, на платформе VK HR Tek"
3. **Не чаще 1 раза в 2–3 абзаца** — итого 3–5 раз в статье
4. **Через сценарии и кейсы**, а не через описание продукта
5. **Конкуренты — нейтрально**, без негативных сравнений
6. **CTA только в конце** — мягкий: "оставьте заявку на демо" или "попробуйте бесплатный модуль"

### Плохо

> VK HR Tek — платформа для кадрового электронного документооборота, которая позволяет автоматизировать 30+ типов документов...

### Хорошо

> Когда у тебя 6500 человек в 1900 точках по стране, курьерская доставка документов — катастрофа. Гемотест выбрал VK HR Tek, потому что нужна была интеграция с 1С и мобильное приложение для сотрудников в лабораториях. Результат — экономия 75% бюджета КДП.

## 10. Чек-лист качества SEO-статьи

Перед показом статьи пользователю проверь:

- [ ] H1 содержит primary keyword
- [ ] Meta title до 60 символов с ключом
- [ ] Meta description 150–160 символов с ключом и CTA
- [ ] Primary keyword в первых 100 словах
- [ ] Плотность primary keyword 1–2%
- [ ] H2 содержат secondary keywords
- [ ] FAQ-секция есть (3–5 вопросов)
- [ ] Таблицы и/или списки есть
- [ ] VK HR Tek не в первых двух абзацах
- [ ] VK HR Tek упоминается через сценарии/кейсы
- [ ] Есть конкретные кейсы с цифрами
- [ ] Нет запрещённых фраз
- [ ] Нет прямой рекламы
- [ ] CTA только в конце
- [ ] Объём соответствует заданному
