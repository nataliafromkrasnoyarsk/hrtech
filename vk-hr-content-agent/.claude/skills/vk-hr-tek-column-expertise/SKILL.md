# Экспертиза для колонок VK HR Tek

> **Зависимость:** Этот скилл работает совместно с `vk-hr-tek-knowledge` (общая база знаний).
> Здесь — только формат-специфичная экспертиза для написания колонок.
> Продукт, модули, спикеры, кейсы, данные, нарративы, tone of voice — в общей базе.

---

## 1. Типы колонок

### Трендовый анализ
**Структура:** Hook → Контекст → 3–5 трендов → Российский угол → Прогноз
**Когда:** Тема связана с развитием рынка, технологий, регуляторики
**Примерный объём:** 1500–2500 слов

### Контринтуитивный взгляд
**Структура:** Bold claim → Общепринятое мнение → Доказательства → Нюансы → Выводы
**Когда:** Есть возможность оспорить устоявшееся мнение с данными
**Примерный объём:** 1200–2000 слов

### Из опыта внедрений
**Структура:** Ситуация → Подход → Результаты → Уроки → Применение
**Когда:** Есть реальный кейс с цифрами и конкретикой
**Примерный объём:** 1500–2500 слов

### Стратегический комментарий
**Структура:** Событие → Контекст → Позиция автора → Что это значит для рынка → Прогноз
**Когда:** Реакция на значимое событие в HR-сфере или законодательстве
**Примерный объём:** 800–1500 слов

### Методология / How-To
**Структура:** Проблема → Фреймворк → Шаги → Подводные камни → Метрики
**Когда:** Можно дать практический алгоритм действий
**Примерный объём:** 1500–3000 слов

## 2. Формулы для колонок

### Формула открытия колонки

```
[Яркая статистика] + [Неочевидный вывод] + [Почему это важно именно сейчас]

Пример: "80% российских компаний до сих пор подписывают кадровые
документы на бумаге. При этом рынок КЭДО растёт на 80% в год.
Парадокс? Нет — точка перелома."
```

**Альтернативные формулы:**

1. **Вопрос + контекст:**
   > Сколько стоит бумажный кадровый документ? Не только бумага и чернила — а логистика, хранение, потеря, пересылка. Мы посчитали — и результат удивил.

2. **Сценарий + масштаб:**
   > Представьте: 6500 сотрудников, 1900 лабораторий, три часовых пояса. Каждый кадровый документ — курьером. Так работал Гемотест до перехода на КЭДО.

3. **Парадокс:**
   > 89% компаний хотят цифровизировать HR. Но 80% до сих пор не перешли на КЭДО. Не потому что не хотят — потому что боятся.

### Формула аргументации

```
[Проблема бизнеса] → [Данные рынка] → [Кейс реальной компании] → [Практический вывод]
```

Каждый аргумент в колонке должен проходить через эту цепочку. Без данных — голословие. Без кейса — теория. Без вывода — нет ценности для читателя.

### Формула кейса внутри колонки

```
[Компания + масштаб] → [Конкретная боль] → [Что сделали] → [Цифры результата]
```

Кейс внутри колонки — не отдельная история, а аргумент. 3–5 предложений, не больше.

### Формула заключения

```
[Синтез ключевого тезиса] → [Практический совет для читателя] → [Взгляд вперёд]
```

Заключение не должно повторять введение. Это самостоятельная мысль, которая вытекает из аргументации.

## 4. Чеклист качества колонки

### Контент
- [ ] Цепляющий hook в первых 2–3 предложениях
- [ ] Чёткий тезис в первых 3 абзацах
- [ ] Российский контекст (ФЗ-377, Минтруд, регуляторика)
- [ ] Данные из исследований с указанием источников
- [ ] Реальные кейсы компаний с цифрами
- [ ] Практические выводы для читателя
- [ ] НОЛЬ прямого продвижения VK HR Tek

### Соответствие изданию
- [ ] Правильный объём (слова/знаки)
- [ ] Подходящий тон (формальный/разговорный)
- [ ] Нужная техническая глубина
- [ ] Правильное форматирование

### Соответствие продукту
- [ ] Экспертиза автора ощущается (знание рынка КЭДО изнутри)
- [ ] VK HR Tek НЕ упоминается в тексте (для Forbes) или упоминается органично
- [ ] Нет маркетинговой риторики
- [ ] Нет сравнений «мы лучше HRlink/Directum»
- [ ] Конкуренты упомянуты нейтрально (если упомянуты)

### Язык
- [ ] Правильный регистр (деловой/разговорный)
- [ ] Единообразная терминология (КЭДО, ЭП, ФЗ-377)
- [ ] Активный залог
- [ ] Типографика: — (не -), «» (не ""), ё
- [ ] Нет корпоспика

## 6. Типичные доработки колонок

| Запрос | Что делать |
|--------|-----------|
| Сильнее hook | Предложить 2–3 альтернативных открытия по формулам выше |
| Больше данных | Подобрать статистику из общей базы знаний |
| Больше кейсов | Добавить кейс клиента с цифрами из общей базы |
| Мягче тон | Уменьшить категоричность, добавить оговорки |
| Короче | Убрать второстепенные аргументы, оставить самые сильные |
| Другой угол | Переструктурировать вокруг нового тезиса |
| Больше законодательства | Усилить регуляторный контекст (ФЗ-377, ФЗ-407) |
| Сменить спикера | Адаптировать голос под нового спикера из общей базы |
